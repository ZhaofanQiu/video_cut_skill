# Architecture Decision Records (ADRs)

This directory contains Architecture Decision Records (ADRs) that document significant architectural decisions made in the Video Cut Skill project.

## What is an ADR?

An Architecture Decision Record (ADR) captures an important architectural decision made along with its context and consequences. ADRs help teams understand:
- Why certain decisions were made
- What alternatives were considered
- What trade-offs were accepted

## Status Definitions

- **Proposed**: Under discussion, not yet decided
- **Accepted**: Approved and implemented
- **Deprecated**: Was accepted but is no longer relevant
- **Superseded**: Replaced by a newer ADR (link provided)

## Writing New ADRs

When creating a new ADR:

1. Use the next sequential number
2. Follow the template in existing ADRs
3. Include:
   - Status and date
   - Context (why the decision was needed)
   - Decision (what was decided)
   - Consequences (positive and negative)
   - Alternatives considered
4. Update this index
